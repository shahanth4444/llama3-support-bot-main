# Contributing to Llama 3 Support Bot

Thank you for your interest in contributing! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit with meaningful messages following conventional commits
6. Push to your fork
7. Submit a pull request

## Conventional Commits

Please use conventional commits for commit messages:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `test:` for tests
- `build:` for build configuration
- `ui:` for UI changes
- `refactor:` for code refactoring
- `perf:` for performance improvements

Example: `feat: add RAG integration for policy grounding`

## Testing

- Write tests for new features
- Ensure all tests pass locally before submitting PR
- Test with both zero-shot and one-shot prompting methods

## Documentation

- Update relevant documentation when making changes
- Include examples in docstrings
- Keep README.md updated with new features

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and modular

## Questions?

Feel free to open an issue for questions or discussions.
